Hexo博客在W11上安装及部署到Github记录

2022-07-04 642 字
hexo

基础环境和工具

环境系统：W11

基础工具：记事本（W11自带），cmd（WIN+R，输入cmd）

Hexo博客本地安装流程

Hexo安装前置

安装Node.js

浏览器搜索Node.js,找到官网http://nodejs.cn/，下载符合自己电脑的版本，选择安装位置，然后一直点击“下一步”，安装完成。

node -v    /*显示node版本。*/
安装淘宝的cnpm管理器
npm install -g cnpm --registry=https://registry.npm.taoboo.org

cnpm -v     /*显示cnpm版本。*/
利用cnpm安装hexo
cnpm install -g hexo-cli   /*-g 全局安装*/

hexo -v  /*显示hexo版本。
           可以注意到版本信息中其基于node.js
         */
备注

hexo需要Node.js生成

E:\>clean                                     /*clean在W11中无法使用*/
E:\>sudo su                                  /*sudo在W11中无法使用，也不需要输入sudo su来切换账                                               户，一般登录账户为管理员账户*/

<– –>


hexo正式安装

选择安装博客所在的本地位置
C:\Users>     /*这代表路径，当前处于C盘下Users文件夹）*/
C:\Users>cd ..           /*cd..代表返回上一层*/

C:\>E:                   /*返回到C：\> 时，输入E：，就可以切换到E盘（前提你有E盘）*/

E:\>cd coding            /*输入cd *（+ 想访问的文件夹名字）*/

E:\coding>mkdir blog     /*这样在E盘coding这个文件夹下会出现一个叫blog的文件夹（可以叫其他名字）*/
初始化博客
E:\>cd coding\blog

E:\coding\blog>hexo init
INFO  Cloning hexo-starter https://github.com/hexojs/hexo-starter.git
[32mINFO [39m Install dependencies
npm WARN config global `--global`, `--local` are deprecated. Use `--location=global` instead.
INFO  Start blogging with Hexo!      /*代表成功*/
启动和退出博客
E:\coding\blog>hexo s                    /*s是start缩写*/
INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.

在浏览器中输入， 则显示如下网页

键盘按Ctrl+C退出，至此，博客在本地已经安装好了，文章的书写见其他文章。

博客部署到Github上

github创建仓库

登录GitHub，头像旁边的+号，新建一个repository，我已经创建过一个，所以会报错，点击下面的Create repository 按键就可以创建

blog目录下安装git插件

E:\coding\blog> cnpm install --save hexo-deployer-git

设置_config.yml文件

W11的资源管理器（快捷键WIN+E）下进入E:\coding\blog这个目录下面，用记事本打开，

在最下面type: ‘’后修改成下述样式

deploy:
type: git
repo: https://github.com/zimu-13/zimu-13.github.io.git
branch: master

repo后是仓库HTTPS的连接网址

安装git，且设置密钥

该部分见git

部署

E:\coding\blog>hexo d
